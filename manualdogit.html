O que é Git?

O Git é um sistema de controle de versão de código-fonte aberto. Ele facilita esse tipo de colaboração no projeto por meio do controle de versão distribuído de arquivos que residem em repositórios. Em essência, o Git possibilita a integração de fluxos de trabalho realizados por vários colaboradores com o passar do tempo, para um determinado repositório.

Fonte: https://learn.microsoft.com/pt-br/contribute/content/git-github-fundamentals

O que é um sistema de controle de versão?

Sempre que desenvolvedores criam um novo projeto eles continuam criando atualizações no código base. Mesmo depois de o projeto ser lançado é comum a atualização de versões, correção de bugs, adição de novas ferramentas, etc.

O sistema de controle de versão ajuda a acompanhar as mudanças feitas no código base. E mais, ele também registra quem efetuou a mudança e permite a restauração do código removido ou modificado.

Não há códigos sobrescritos uma vez que o Git salva múltiplas cópias no repositório.

Fonte: https://www.hostinger.com.br/tutoriais/o-que-github

Para que serve o Git?

O objetivo principal objetivo do Git é permitir o controle de histórico e versão desses projetos, melhorar o trabalho em time e o fluxo de trabalho, proporcionar a segurança dos seus arquivos e outras tantas vantagens faladas nesse post.

Onde o Git pode ser usado?

Pode ser usado em todo e qualquer projeto que tenha arquivos de diferentes tipos, podendo ser código, texto, imagens, vídeos, áudios, entre outros.

Fonte: https://blog.betrybe.com/git/

Quais as funções do Git?

As funções do Git podem ser:

  • Gerenciar projetos com repositórios,
  • Clonar um projeto para trabalhar em uma cópia local,
  • Controlar e rastrear alterações com preparação e confirmação
  • Permitir trabalhar em diferentes partes e versões de um projeto com os comandos branch e merge,
  • Extrair a versão mais recente do projeto para uma cópia local e
  • Enviar atualizações locais para o projeto principal.

Como trabalhar com o Git?

  • Inicialize o Git em uma pasta, tornando-o um Repositório,
  • O Git agora cria uma pasta oculta para acompanhar as alterações nessa pasta,
  • Quando um arquivo é alterado, adicionado ou excluído, ele é considerado modificado,
  • Você seleciona os arquivos modificados que deseja preparar,
  • Os arquivos preparados são confirmados, o que solicita que o Git armazene um instantâneo permanente dos arquivos,
  • O Git permite que você veja o histórico completo de cada commit,
  • Você pode reverter para qualquer commit anterior e
  • O Git não armazena uma cópia separada de cada arquivo em cada commit, mas monitora as alterações feitas em cada commit.
